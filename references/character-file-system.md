## Why Individual Files

长篇续写中，最大的挑战之一是**角色走样**。随着续写越来越长，模型的上下文窗口无法同时容纳所有角色信息和已写内容，导致：
- 角色说话方式逐渐偏离原文
- 角色行为不符合性格设定
- 角色关系发展失去一致性
- 忘记角色之前的状态和经历

**解决方案：为每个主要角色维护一个独立的 .txt 文件。**

这个文件是角色的"记忆"：
- 写作前读取 → 确保了解角色
- 写作后更新 → 记录角色的新发展
- 反思时核对 → 检查是否走样

## File Structure

每个角色文件包含以下区块：

```
=== [角色名] 角色档案 ===

【基本信息】
姓名、年龄、外貌、身份等客观信息

【性格特征】
核心性格、优缺点、欲望、恐惧

【语言风格】
说话特点、常用词汇、语气、禁忌

【行为模式】
习惯动作、冲突反应、决策方式

【原文中的关键情节】
角色在原文中的重要事件和表现

【人物关系】
与其他角色的关系动态

【角色弧线状态】
当前状态、正在经历的变化、预期方向

【续写更新日志】
每次续写后记录的新发展（按章节记录）
```

填写时注意：性格用具体描述（"坚韧"比"好"有用），语言风格记录具体模式和禁忌，关键情节写事件+表现+变化，续写更新日志每章必更。

## Character Evolution

角色不是静态的。随着故事发展，角色的性格、语言、行为都会发生变化。角色文件必须能追踪这种演变。

### 核心原则

**角色可以变，但变化必须合理。**

- ✅ 经历重大打击后变得沉默寡言 — 有事件驱动，合理
- ❌ 上一章还懦弱，这一章突然勇敢 — 没有铺垫，不合理
- ✅ 信任被背叛后开始多疑 — 有因果关系，合理
- ❌ 用词风格突然从口语变书面语 — 没有原因，不合理

### 哪些会变，哪些不变

**容易变化的：**
- 情感状态（开心/悲伤/愤怒）
- 对特定人物的态度
- 行为习惯（经历创伤后可能改变）
- 处事方式（成熟/圆滑）
- 信任程度

**缓慢变化的：**
- 核心价值观
- 语言风格（除非有重大事件冲击）
- 基本性格特质
- 思维模式

**几乎不变的：**
- 核心人格（内向/外向的底色）
- 深层欲望和恐惧
- 说话的基本节奏和习惯

### 如何在角色文件中追踪演变

**方法1：在【续写更新日志】中标注变化**

```
【续写更新日志】
- 第3章：发现好友背叛，信任感受到严重打击
  → 性格变化：从轻信变得多疑
  → 语言变化：对话变少，更多沉默
  → 关系变化：与李明关系从信任→怀疑
- 第5章：在危机中被陈雨搭救，开始重新信任
  → 性格变化：多疑有所缓解，但保留警惕
```

**方法2：在对应字段中更新当前状态**

当角色发生实质性变化时，直接更新对应的字段：

```
【性格特征】
- 核心性格：[原始描述] → [当前描述，标注变化]

【语言风格】
- 说话特点：[原始描述] → [当前描述，标注变化]

【行为模式】
- 面对冲突时的反应：[原始] → [当前，标注变化]
```

**方法3：在【角色弧线状态】中记录演变轨迹**

```
【角色弧线状态】
- 原文结束时：天真、轻信、乐观
- 第1-10章：经历背叛，开始怀疑一切
- 第11-20章：在新朋友帮助下，学会选择性信任
- 当前状态：不再天真，但仍保有善意
```

### 反思时的演变检查

反思检查时（arc完成或15章兜底），对每个角色问：
1. 这个角色和原文结束时相比，变了多少？
2. 变化是由哪些事件驱动的？
3. 变化过程是否有足够的铺垫？
4. 变化后的言行是否自洽？
5. 核心特质是否被保留？

## Creation Process

### Step 1: 识别主要角色

从原文中识别需要创建独立文件的角色：
- **必须创建**: 主角、主要反派、核心配角
- **建议创建**: 戏份较多的次要角色
- **可选**: 用户特别关注的角色
- **不需要**: 龙套角色、仅提及的角色

### Step 2: 提取信息

从原文中为每个角色提取：
1. 基本信息（直接描写）
2. 性格特征（从行为、对话、心理描写推断）
3. 语言风格（从对话中提取）
4. 行为模式（从关键场景中总结）
5. 关键情节（列出角色参与的重要事件）
6. 人物关系（与其他角色的互动）
7. 弧线状态（角色当前的发展阶段）

### Step 3: 创建文件

为每个角色创建 `characters/[角色名].txt` 文件，按模板填写。

**注意：**
- 只写原文中明确展现或强烈暗示的信息
- 不确定的信息标注 [推测]
- 保持客观，不加入续写期望

### Step 4: 用户确认

将角色文件展示给用户，确认：
- 信息是否准确？
- 是否遗漏重要特征？
- 性格解读是否正确？

## Reading Protocol

### 何时读取

1. **每章写作前** — 读取该章出场角色的文件
2. **反思检查时** — 读取所有活跃角色的文件
3. **用户询问时** — 读取相关角色的文件

### 读取策略

**按需读取，而非全部加载。**

长篇续写可能有 8+ 个角色文件，全部加载会浪费上下文空间。

```
写第5章 → 读取第5章出场的3个角色文件
写第6章 → 读取第6章出场的2个角色文件（可能与第5章不同）
```

### 读取重点

每次读取时重点关注：
1. 【语言风格】— 确保角色说话方式正确
2. 【行为模式】— 确保角色行为符合性格
3. 【续写更新日志】— 了解角色最近的状态
4. 【人物关系】— 确保关系处理正确

## Update Protocol

### 何时更新

**每章写完后立即更新。** 不要攒到后面一起更新。

### 更新内容

在角色文件的【续写更新日志】中添加新条目：

```
【续写更新日志】
（原文中的记录）
- 第X章：[状态变化、关系变化、新决定、新发现等]
- 第X章：...
```

### 更新规则

1. **只记录变化** — 如果角色在某章没有实质性变化，不强制记录
2. **具体而非笼统** — "与李明发生争执，关系恶化" 比 "有冲突" 有用
3. **记录因果** — "因为发现真相，决定离开" 比 "离开" 有用
4. **及时更新关系** — 如果角色关系发生变化，同时更新【人物关系】区块
5. **更新弧线状态** — 如果角色有重大发展，更新【角色弧线状态】区块

### 更新示例

写完第3章后，角色"林远"的更新：

```
【续写更新日志】
- 第1章：发现父亲留下的信件，得知家族秘密
- 第2章：与陈雨讨论信件内容，决定调查真相
- 第3章：在废弃工厂找到线索，遭遇不明人物袭击，左臂受伤
  → 更新：人物关系 - 与陈雨的信任加深
  → 更新：角色弧线 - 从被动接受转为主动追查
```

## File Naming & Organization

### 命名规则

- 使用角色名作为文件名：`characters/林远.txt`
- 英文角色使用英文名：`characters/john.txt`
- 如果有重名，加后缀区分：`characters/林远-将军.txt`
- 文件统一使用 .txt 格式（纯文本，兼容性最好）

### 目录结构

```
characters/
├── 林远.txt          # 主角
├── 陈雨.txt          # 女主角
├── 张明.txt          # 反派
├── 李老.txt          # 导师角色
└── 王五.txt          # 配角
```

### 备份策略

如果续写很长（50章+），建议定期备份角色文件：
```
characters/
├── 林远.txt
├── 林远-backup-ch20.txt    # 第20章时的备份
└── ...
```
