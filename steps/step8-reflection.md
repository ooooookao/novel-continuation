# Step 8: Arc 完成反思检查（子 agent）

**每个 arc 写完后，执行一次全面反思检查。** 由子 agent 执行，输出报告给主 agent。

## 执行清单

### 触发条件（满足任一）
- [ ] 当前 arc 最后一章写完 → 立即触发
- [ ] 距上次反思 ≥ 15 章且 arc 未结束 → 兜底触发

### 执行
- [ ] 准备材料：风格档案 + 反思范围章节 + 原文5章 + 角色文件 + 大纲
- [ ] 调用反思子 agent（失败则主 agent 做简化版：只查最近5章，只查维度1-6）
- [ ] 检查维度 1-6（必做）：字数 / 文风 / 人物 / 情节 / 整体 / 节奏
- [ ] 检查维度 7（必做）：内容重复检测（6 类重复模式）
- [ ] 检查维度 8-10（仅 arc 完成时）：Arc 结构 / 伏笔收束 / 角色弧
- [ ] 保存报告 → `reports/reflection-arc{N}.md` 或 `reports/reflection-chX-chY.md`
- [ ] 向用户展示关键发现
- [ ] 根据报告调整：字数补写 / 风格约束 / 角色文件 / 大纲 / 重复模式注入禁止项

### 反思后
- [ ] 生成 checkpoint → `deltas/checkpoint-arc{N}.md`
- [ ] 更新 `progress.md` 中的【Arc 追踪】和【上次反思章节】
- [ ] **→ 回到 Step 7 继续写下一章**

## 触发条件（满足任一即触发）

1. **当前 arc 的最后一章写完** → 立即触发反思
2. **距上次反思已过 15 章且当前 arc 仍未结束** → 也触发一次中间反思（兜底机制，防止长 arc 失控）

主 agent 在每章写完后检查：`当前章节序号 - 上次反思章节序号`，若 ≥ 15 且当前 arc 未完结，则触发中间反思。

**中间反思 vs arc 完整反思：**
- 中间反思（兜底触发）：范围为上次反思至今的所有章节，重点检查风格漂移和节奏单调问题
- Arc 完整反思：范围为整个 arc 所有章节，增加 arc 级别检查（伏笔收束、角色弧完成度、arc 结构完整性）

## 主 agent：准备上下文

调用子 agent 前，准备以下材料：

1. 读取 `style-profile.md` — 风格档案全文
2. 读取本次反思范围内的所有续写内容（`chapters/chapter-XXX.txt`）
3. 从原文中随机抽取 5 章的内容
4. 读取本次反思范围涉及的角色的角色文件
5. 读取 `outline-continuation/` 中对应 arc 的大纲

## 主 agent：调用子 agent 反思

将材料组装成 prompt，调用子 agent：

```
对以下续写章节进行全面反思检查。模板参考 [assets/templates/reflection-report.md](../assets/templates/reflection-report.md)。

【反思类型】[arc完成反思 / 中间兜底反思]
【反思范围】第X章 - 第Y章（共Z章）
【对应Arc】[arc名称/编号，中间反思时写"进行中"]
【风格档案】[style-profile.md 全文]
【续写章节】[反思范围内所有章节内容]
【原文5章】[随机抽取的原文章节]
【角色文件】[涉及角色的约束卡]
【目标字数/章】[X字]
【Arc大纲】[该arc的大纲，中间反思时为当前arc大纲]

检查维度：
1. 字数：逐章统计，标注不达标项及扩充方向
2. 文风双轨对比：A轨对比风格档案，B轨对比原文5章
3. 人物一致性：基础检查+演变追踪
4. 情节连贯性：主线/支线/伏笔/逻辑
5. 整体评估：最满意/最需改进/后续方向
6. 节奏曲线：逐章紧张度(1-10)+情感基调，用█/░画曲线图，识别连续低潮/高潮/单调问题
7. 内容重复检测：扫描反思范围内所有章节，检测以下重复模式：
   a. 场景描写重复：相似的环境/场景描写是否换汤不换药（如同一个地点多次描写几乎相同、不同地点用了相同的描写套路）
   b. 情绪描写重复：是否反复使用相同的词汇/句式表达情绪（如每次都"心头一紧"、每次都"沉默片刻"）
   c. 对话模式重复：对话的节奏/结构是否雷同（如总是"提问→沉默→回答"、总是"反驳→解释→接受"）
   d. 转场方式重复：章节/场景之间的过渡手法是否千篇一律
   e. 感官描写套路：是否总用同一感官优先（如每次都是视觉描写打头，从不用听觉/嗅觉）
   f. 角色动作重复：某角色是否反复做同一个标志性小动作（非刻意设定的习惯，而是写作偷懒）
   对每个命中项标注：重复内容、出现次数、涉及章节、严重程度（轻微/明显/严重）

仅 arc 完成反思时额外检查：
8. Arc 结构完整性：起承转合是否到位，高潮是否有力度
9. 伏笔收束：本 arc 应揭晓的伏笔是否全部揭晓，有无遗漏
10. 角色弧完成度：本 arc 角色的成长/转变是否完成

输出：按反思报告模板格式。
```

**反思报告模板**: [assets/templates/reflection-report.md](../assets/templates/reflection-report.md)

## 主 agent：处理反思结果

子 agent 返回反思报告后，主 agent：

1. 将报告保存到 `reports/reflection-arc{N}.md`（arc 完整反思）或 `reports/reflection-chX-chY.md`（中间反思）
2. 向用户展示关键发现（字数问题、偏差、问题、建议）
3. 如果发现问题，根据报告调整：
   - **字数不达标** → 调用子 agent 补写：将扩充建议作为 prompt，让子 agent 对未达标章节进行扩写，输出补写片段，主 agent 插入到对应位置。扩写后再次校验字数，最多重试 3 次，仍不达标则向用户报告
   - 风格偏差 → 在后续章节的写作 prompt 中加强风格约束
   - 角色走样 → 更新角色文件
   - 情节漏洞 → 调整后续大纲
   - **内容重复** → 将重复模式列表注入后续章节的写作 prompt 的【强制要求】区块，列为禁止项；严重重复的章节段落调用子 agent 重写
   - **伏笔遗漏**（arc 反思特有）→ 在后续章节补设揭晓或标记到 threads.md
