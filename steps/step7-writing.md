# Step 7: 逐章写作（子 agent）

大纲确认后，按章节顺序写作。**每章由子 agent 执行**，主 agent 负责准备上下文和更新状态。

## 断点续写恢复

新会话恢复时，跳过 Step 1-6，执行以下恢复流程后进入写作循环：

- [ ] 读取 `progress.md` — 确认写到第几章、当前哪个 arc
- [ ] 读取最近的 checkpoint + 之后的 deltas — 重建完整状态
- [ ] 检查【待处理反馈】— 有则先处理再继续写
- [ ] 读取当前 arc 的大纲（`outline-continuation/arc-XX.md`）
- [ ] 读取本章出场角色的角色文件（`characters/`）
- [ ] 读取 `style-profile.md` 的【标志性特征】和【续写对照清单】
- [ ] 读取 `world-facts.txt` 相关设定
- [ ] 读取前一章结尾 200-300 字
- [ ] 询问用户强制提示词（展示上一批次的，问是否沿用或修改）
- [ ] **→ 进入下方写作循环**

## 每章执行清单

### 写作前准备
- [ ] 读取 `progress.md` — 当前进度、强制提示词
- [ ] 读取本章出场角色的角色文件
- [ ] 读取本章大纲
- [ ] 读取 `style-profile.md` 的【标志性特征】和【续写对照清单】（不要读全文）
- [ ] 读取 `world-facts.txt` 相关设定
- [ ] 读取前一章结尾 200-300 字
- [ ] 询问用户强制提示词（每批开始时问一次，批次内沿用）

### 写作
- [ ] 组装 prompt 调用写作子 agent
- [ ] 子 agent 返回章节正文
- [ ] 子 agent 调用失败？→ 重试 3 次（0s/5s/15s），仍失败 → 主 agent 自己写

### 字数校验（强制，不可跳过）
- [ ] 统计实际字数
- [ ] 与目标字数对比（≥ 90% 为达标）
- [ ] 不达标？→ 调用扩写子 agent（失败则主 agent 自己扩写）
- [ ] 扩写后再次校验，最多重试 3 次
- [ ] 3 次仍不达标 → 熔断，向用户报告

### 质量审查（强制，不可跳过）
- [ ] 调用质量审查子 agent（失败则主 agent 自己审，评分标准不变）
- [ ] 5 维评分：OOC / 世界观 / 逻辑 / 风格 / 不重复
- [ ] ≥ 9.0 → 自动通过
- [ ] 7.0-8.9 → 向用户展示，用户决定
- [ ] < 7.0 → 退回重写，最多 2 次
- [ ] 反 AI 味检测：≤2项通过 / 3-4项修正 / ≥5项退回
- [ ] 3 次重写仍不达标 → 熔断，向用户报告

### 保存与更新（全部通过后，按顺序执行）
- [ ] 保存章节 → `chapters/chapter-XXX.txt`
- [ ] 生成 delta 文件 → `deltas/delta-chXXX.md`
- [ ] 更新 `progress.md`
- [ ] 更新 `threads.md`（伏笔账本）
- [ ] 更新出场角色的角色文件
- [ ] 更新 `world-facts.txt`（如有新设定）
- [ ] 更新大纲（如有偏差）
- [ ] 本章是当前 arc 最后一章？→ 生成 checkpoint → `deltas/checkpoint-arc{N}.md`

### 写完后检查
- [ ] 检查是否触发反思：arc 最后一章？距上次反思 ≥ 15 章？
- [ ] 触发 → 进入 Step 8
- [ ] 未触发 → 继续写下一章（回到"写作前准备"）

## 主 agent：准备上下文

每章写作前，主 agent 准备子 agent 需要的上下文：

1. 读取 `progress.md` — 确认当前进度、待处理反馈
2. 读取该章出场角色的角色文件（`characters/[角色名].txt`）— 只读本章需要的
3. 读取该章的大纲 — 从对应的弧线文件中读取
4. 读取 `style-profile.md` 的【标志性特征】和【续写对照清单】— 不要读全文
5. 读取 `world-facts.txt` 中相关设定
6. 读取前一章的结尾 200-300 字 — 确保衔接

## 主 agent：询问用户强制提示词

上下文准备完毕后、调用子 agent 写作前，**必须询问用户**：

```
即将开始写第X章，你有什么需要加入的强制提示词吗？

例如：
- 本章必须包含某个情节或场景
- 某个角色必须出场/不能出场
- 必须描写某个特定事件
- 情绪基调要求（如：压抑、轻松、紧张）
- 字数要求调整（如：本章需要特别长/短）
- 其他任何强制性要求

如果没有特殊要求，回复"没有"即可继续写作。
```

将用户的强制要求记录到 `progress.md` 的【当前批次强制提示词】中，并合并到子 agent 的写作 prompt 的【写作要求】区块。**每批写作开始时询问一次**（如开始写新的一批章节、或断点续写恢复时），批次内沿用同一组强制提示词，直到用户主动修改。不要每章都问。

## 主 agent：调用子 agent 写作

将准备好的上下文组装成 prompt，调用子 agent：

```
你是一个小说续写助手。请续写第X章。

【风格要点】[style-profile.md 的标志性特征和对照清单]
【本章大纲】[从弧线文件中提取]
【出场角色】[角色约束卡，仅本章出场角色]
【世界观】[world-facts.txt 相关条目，精简]
【活跃伏笔】[threads.md 活跃伏笔，仅含本章及之后揭晓的]
【应揭晓伏笔】[threads.md 预计本章揭晓的，无则省略]
【前一章结尾】[最后200-300字]

【强制要求】
- 字数必须≥X字（最低X×90%），不足退回重写
- 开头自然衔接前一章结尾
- 严格保持风格一致性，角色言行符合约束卡
- 按大纲推进情节，每章结尾留钩子
- 按场景分段，场景间空一行

直接输出章节正文，不加说明。
```

子 agent 返回章节正文后，进入**字数校验**环节。

## 主 agent：字数校验（强制）

**每章写完后必须立即校验字数，不达标则扩写，绝不跳过。**

1. **统计实际字数**：对子 agent 返回的章节正文进行字数统计（中文按字符数，英文按词数）
2. **与目标字数对比**：目标字数来自 Step 3 确认的参数（记录在 `progress.md` 中）
3. **判定标准**：实际字数 ≥ 目标字数 × 90% 为达标，否则不达标
4. **不达标处理**：立即调用扩写子 agent，直到达标为止

**扩写子 agent 的 prompt：**
```
以下章节字数未达标，请扩充。

【风格要点】[style-profile.md 的标志性特征，仅关键点，不超过10行]
【当前章节】[章节正文]
【字数】当前X字，目标≥Y字（差W字）

扩充方向：心理描写、环境描写、感官细节、角色互动。禁止水词/重复/偏离大纲。
输出完整扩写后章节正文，不加说明。
```

5. **循环校验**：扩写后再次统计字数，仍未达标则继续扩写。最多重试 3 次，若 3 次后仍未达标，向用户报告情况并请求指示
6. **达标后进入质量审查**：字数达标后，执行下方的质量审查流程

## 主 agent：质量审查（强制）

**每章字数达标后、保存前必须通过质量审查。不通过则退回重写。**

调用质量审查子 agent，执行 5 维评分 + 反 AI 味检测：

```
审查以下章节。评分标准参考 [references/quality-scoring.md](../references/quality-scoring.md)。

【风格要点】[style-profile.md 标志性特征，精简]
【角色约束卡】[本章出场角色的约束卡]
【世界观】[world-facts.txt 相关条目，精简]
【活跃伏笔】[threads.md 活跃伏笔]
【本章大纲】[应推进的情节和应揭晓的伏笔]
【待审查章节】[章节正文]

输出格式（严格遵守，不要输出其他内容）：
【评分】OOC:X/10 | 世界观:X/10 | 逻辑:X/10 | 风格:X/10 | 不重复:X/10 | 总分:X.X/10
【扣分】每项扣分原因，一句话
【反AI味】命中项序号+位置，无则写"无命中"
【判定】通过/建议修改/退回重写
【修改】具体修改要求（如有）
```

**评分判定规则：**
- **≥ 9.0 分**: 自动通过，保存章节
- **7.0 - 8.9 分**: 向用户展示评分报告和修改建议，用户决定是否接受或要求修改
- **< 7.0 分**: 自动退回，调用子 agent 按修改要求重写，重写后重新走字数校验 + 质量审查流程。最多退回 2 次，仍低于 7.0 则向用户报告

**反 AI 味命中处理：**
- 命中 ≤ 2 项: 通过，但在评分报告中标注，供用户参考
- 命中 3-4 项: 调用子 agent 针对性修正命中项，修正后重新评分
- 命中 ≥ 5 项: 退回重写

**质量审查通过后进入更新流程。**

## 并行写作（批量模式）

一次写 5 章时，使用**并行写 + 审润**流程：

```
5个子agent并行写作（各写1章）
    ↓
第6个子agent审润（检查衔接，必要时修改）
    ↓
主agent保存章节 + 更新状态
```

**并行写作条件：**
- 大纲已经详细到场景级别
- 每章的开头基于大纲描述，而非严格依赖前一章的实际结尾
- 每章结尾留钩子，但不需要和下一章的实际内容完全对接

**5 个子 agent 的 prompt 中：**
- 用大纲中该章的"①[场景描述]"作为开头引导，而非前一章的结尾
- 每章独立完成，互不等待
- 合并用户强制提示词到【写作要求】区块（同单章写作流程）

**审润 agent 的任务：**
- 读取 5 章内容
- 检查相邻章节的衔接：
  - **同一主线**（如 A线第33章 → A线第34章）：检查开头是否能接上一章结尾，如果不能，修改开头过渡段（200-300字）
  - **不同主线**（如 A线第33章 → B线第34章）：不修改，只检查风格一致性
- 输出审润后的 5 章内容 + 一份简要衔接报告

**审润 agent 的 prompt：**
```
以下5章是并行写作的，检查并润色衔接。

【5章内容】[第X/Y/Z/A/B章内容]
【主线归属】第X章:A线 | 第Y章:A线 | 第Z章:B线 | ...

规则：
- 同主线相邻章：检查衔接，断裂则修改下一章开头200-300字
- 不同主线相邻章：不改衔接，只查风格一致性
- 全部：查风格/角色/矛盾一致性

输出：修改后的完整章节（无修改则原样）+ 简要衔接报告。
```

审润完成后，主 agent 对每章执行**字数校验** + **质量审查**（同单章写作流程：统计字数 → 不达标则扩写 → 达标后质量评分 → 通过后保存）。全部通过后，将 5 章保存到 `chapters/`，然后统一更新状态文件。

**续写技巧**: 参见 [references/continuation-techniques.md](../references/continuation-techniques.md)

## 写作模式变体

除默认的**标准模式**和**批量模式**外，还有三种变体。它们都走标准的字数校验 → 质量审查 → 保存流程，差异仅在触发方式和输入：

| 模式 | 触发 | 差异 |
|------|------|------|
| **定向重写** | "重写第 X 章" | prompt 额外包含原章节全文 + 用户反馈；重写后覆盖原文件 |
| **片段续写** | "从这段接着写" + 粘贴片段 | prompt 额外包含用户片段；第一句必须无缝衔接 |
| **全自动** | "自动写到第 X 章" | 循环执行标准流程；每 5 章暂停展示摘要/评分/伏笔/节奏；质量 < 8.0 不暂停直接退回；到终止章节后停止 |
