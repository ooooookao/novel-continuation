# Step 6: 大纲自审（子 agent，两轮）

用户确认大纲后、开始写作前，由子 agent 对大纲进行**两轮审查**：
- **第一轮（粗调）**：抓结构性硬伤，9 维全部扫描但只报严重/明显问题
- **第二轮（细调）**：粗调问题修完后，重新扫描全部 9 维，报所有级别问题（含可选优化）

## 执行清单

### 第一轮：粗调
- [ ] 调用审查子 agent，9 维扫描，**只报告严重和明显问题**（跳过可选优化）
- [ ] 生成审查报告 → 保存到 `reports/outline-review-coarse.md`
- [ ] 向用户展示报告摘要 + 修改方案
- [ ] 用户决定：全部接受 / 部分接受 / 跳过 / 重新调整
- [ ] 按用户选择修改大纲

### 第二轮：细调
- [ ] 调用审查子 agent，9 维扫描，**报告所有级别问题**（含可选优化）
- [ ] 生成审查报告 → 保存到 `reports/outline-review-fine.md`
- [ ] 向用户展示报告摘要 + 修改方案
- [ ] 用户决定：全部接受 / 部分接受 / 跳过
- [ ] 按用户选择修改大纲
- [ ] **→ 进入 Step 7**

## 第一轮：粗调 prompt

```
对以下续写大纲进行**第一轮粗调审查**。只关注结构性硬伤，跳过细节优化建议。

【续写大纲】[用户确认后的完整大纲]
【原文大纲】[outline-original.md]
【角色文件】[所有角色的约束卡]
【风格档案】[style-profile.md 标志性特征]
【续写参数】[章数、每章字数、情节方向]

审查维度（共 9 项，粗调只报严重/明显问题，跳过可选优化）：

1. 情节发展曲线 — 有无严重原地踏步？高潮位置是否根本性不合理？
2. 人物关系变化 — 有无关系突然跳跃（完全无铺垫的转变）？
3. 重复情节检测 — 有无大量重复模式（≥3次相同事件模式）？
4. 伏笔密度与分布 — 有无伏笔完全找不到揭晓安排？分布是否严重失衡？
5. 冲突升级曲线 — 有无连续大量章节（≥5章）只有单一类型冲突？
6. 支线与主线交织 — 有无支线完全没有交汇点（断头支线）？
7. 逻辑自洽性 — 有无时间线矛盾？角色动机与行为根本性冲突？
8. 角色出场均衡 — 有无核心角色完全消失（≥10章无戏份）？
9. 字数可行性 — 有无大量章节（≥5章）明显偏薄或过密？

对每个维度：
- 评分：优/良/中/差（粗调只关注"差"和"中"，"良"和"优"跳过）
- 列出严重/明显问题（标注章节号）
- 每个问题至少给出 1 个修改方案

输出格式严格遵守审查报告模板。报告标题注明"第一轮粗调"。
```

**粗调通过标准**：9 维全部为"良"或"优"。任一维度为"中"或"差"则需要修改后进入第二轮。

## 第二轮：细调 prompt

```
对以下续写大纲进行**第二轮细调审查**。粗调问题已修复，现在扫描所有级别问题。

【续写大纲】[粗调修改后的完整大纲]
【原文大纲】[outline-original.md]
【角色文件】[所有角色的约束卡]
【风格档案】[style-profile.md 标志性特征]
【续写参数】[章数、每章字数、情节方向]
【粗调报告摘要】[粗调发现的问题和修复情况，避免重复报同类问题]

审查维度（共 9 项，细调报告所有级别问题）：

1. 情节发展曲线 — 冲突升级节奏是否合理？高潮位置是否最佳？有无可优化的铺垫？
2. 人物关系变化 — 关系转变是否有足够的事件驱动？过渡是否自然？有无可增强的互动？
3. 重复情节检测 — 是否有相似模式？差异化是否足够？是否服务于主题递进？
4. 伏笔密度与分布 — 分布是否均匀？揭晓节奏是否合理？有无可优化的埋设位置？
5. 冲突升级曲线 — 冲突类型是否多样？后果是否逐级加重？有无可增加的内心冲突/关系冲突？
6. 支线与主线交织 — 支线价值是否足够？交汇点是否有力？有无可优化的交织方式？
7. 逻辑自洽性 — 时间线是否合理？角色动机是否一致？有无细节矛盾？
8. 角色出场均衡 — 戏份分配是否合理？有无角色可增加/减少出场？
9. 字数可行性 — 逐章评估场景数和事件密度能否撑满目标字数？标注偏薄/过密章节。

对每个维度：
- 评分：优/良/中/差
- 列出所有问题（标注章节号，含可选优化）
- 每个问题至少给出 1 个修改方案

输出格式严格遵守审查报告模板。报告标题注明"第二轮细调"。
```

## 审查报告模板

**报告模板**: [assets/templates/outline-review-report.md](../assets/templates/outline-review-report.md)

## 处理审查报告

子 agent 返回审查报告后，主 agent：

1. 将报告保存到 `reports/outline-review-coarse.md`（粗调）或 `reports/outline-review-fine.md`（细调）
2. 向用户展示审查结果摘要：
   - 各维度评分一览
   - 关键问题列表（标注严重程度：必须修改 / 建议修改 / 可选优化）
   - 每个问题的修改方案（至少 1 个，复杂问题给 2-3 个方案供选择）
3. 用户决定：
   - **全部接受方案** → 主 agent 按方案修改大纲
   - **部分接受** → 主 agent 按用户选择的方案修改，其余保留原样
   - **跳过** → 用户认为大纲已足够好
   - **重新调整** → 大幅修改大纲，修改后可选择重新审查
4. 粗调完成 → 进入细调；细调完成 → 进入 Step 7

**两轮审查全部通过后进入写作阶段。**
