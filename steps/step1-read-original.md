# Step 1: 读取原文

## 执行清单

- [ ] 接收原文（粘贴 / 文件路径 / 分次发送）
- [ ] 确认：原文总章数/字数
- [ ] 确认：是全部还是部分
- [ ] 确认：是否有已续写内容
- [ ] 原文 > 5万字？→ 询问用户选择阅读策略（A. 逐章通读 / B. 抽样读取）
- [ ] 每批读完生成摘要 → `summaries/`
- [ ] 完成后向用户展示确认信息

接收用户提供的小说原文。原文可能以以下方式提供：
- 直接粘贴在对话中
- 提供文件路径（.txt / .md / .docx）
- 分多次发送

**需要确认的信息：**
- 原文总共多少章/多少字？
- 这是原文的全部还是部分？
- 之前是否有已经续写过的内容？

## 确认阅读策略

原文超过 5 万字时，**必须询问用户选择阅读策略**：

```
原文共X章/Y万字，超过5万字，需要选择阅读策略：

1. 逐章通读（推荐，信息完整）
   - 子 agent 按顺序读完所有章节
   - 优点：人物关系、伏笔、世界观细节不会遗漏
   - 缺点：耗时较长，子 agent 需要多轮读取

2. 抽样读取（快速）
   - 从开头、结尾、转折点、高潮各抽几章读
   - 每批读完生成摘要，子 agent 读摘要
   - 优点：速度快
   - 缺点：中间章节细节可能遗漏，伏笔可能丢失

你选哪个？
```

**用户选择后，按对应策略执行。**

## 策略 A：逐章通读

按章节顺序分批读入，每批 5-8 章：

1. **按顺序分批读取** — 每批 5-8 章，逐批读完全部章节
2. **每批读完后生成摘要** — 写入 `summaries/batch-01.md`、`summaries/batch-02.md`...
3. **摘要内容**：该批章节的主要事件、新出场角色、伏笔埋设/揭晓、世界观变化、关键对话
4. **全部读完后**，4 个子 agent 基于所有摘要进行提取

**分批读取流程：**
```
第1批：第1-8章 → 摘要写入 summaries/batch-01.md
第2批：第9-16章 → 摘要写入 summaries/batch-02.md
第3批：第17-24章 → 摘要写入 summaries/batch-03.md
...
最后一批：剩余章节 → 摘要写入 summaries/batch-XX.md
```

每批读完后，更新风格档案和角色文件，逐步完善。

## 策略 B：抽样读取

从不同位置抽样，快速建立全局印象：

1. **先读开头 3-5 章** — 建立初始风格印象、主要角色、世界观
2. **再读结尾 2-3 章** — 了解故事结束时的状态、角色最终位置
3. **中间部分抽样读** — 从不同故事弧中各抽 1-2 章，覆盖转折点、高潮、低谷
4. **每批读完后生成摘要** — 写入 `summaries/` 目录，作为后续参考

**分批读取顺序建议：**
```
第1批：开头3-5章 → 风格/人物/世界观初建
第2批：结尾2-3章 → 了解结局状态
第3批：第一个转折点附近2-3章
第4批：故事中段高潮附近2-3章
第5批：第二个转折点附近2-3章
...根据原文长度继续抽样
```

每批读完后，更新风格档案和角色文件，逐步完善。

**抽样策略的风险提示：** 告知用户抽样可能遗漏伏笔和细节，建议原文有复杂伏笔线时选择逐章通读。
